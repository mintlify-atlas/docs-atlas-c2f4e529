The `AudioManager` is an autoload singleton that provides centralized audio playback throughout your game. It manages `AudioResource` assets and creates temporary audio players for each sound.

## Class signature

```gdscript audio_manager.gd:1
extends Node
```

<Note>
  AudioManager is configured as an autoload singleton in `project.godot`. It's accessible globally as `AudioManager` from any script.
</Note>

### play_sound()

Plays a sound effect with automatic cleanup.

```gdscript audio_manager.gd:13
func play_sound(type: AudioResource.SoundEffect)
```

<ParamField path="type" type="AudioResource.SoundEffect" required>
  The sound effect to play from the SoundEffect enum
</ParamField>

**Behavior:**

1. Looks up the AudioResource for the given type
2. Creates a temporary AudioStreamPlayer node
3. Configures volume and randomized pitch from the resource
4. Plays the sound
5. Automatically frees the player when finished

```gdscript audio_manager.gd:13
func play_sound(type: AudioResource.SoundEffect):
    if not sounds_dic.has(type):
        push_error("No Sound Effect Of Type")
    
    var resource : AudioResource = sounds_dic[type]
    var stream_player : AudioStreamPlayer = AudioStreamPlayer.new()
    add_child(stream_player)
    
    stream_player.stream = resource.stream
    stream_player.volume_db = resource.volume
    stream_player.pitch_scale = rng.randf_range(
        resource.pitchRandomnessLower,
        resource.pitchRandomnessHigher
    )
    stream_player.finished.connect(stream_player.queue_free)
    
    stream_player.play()
```

<Warning>
  If the sound type is not found in the dictionary, an error is logged but execution continues. Ensure all sound types are configured in the AudioManager scene.
</Warning>

## Setup

The AudioManager autoload is configured in `project.godot`:

```ini
[autoload]
AudioManager="*res://Scenes/audio_manager.tscn"
```

## Design notes

<Accordion title="Why temporary audio players?">
  Creating a new AudioStreamPlayer for each sound allows multiple instances of the same sound to overlap naturally. The player automatically cleans itself up via the `queue_free()` callback when playback finishes.
</Accordion>

<Accordion title="Performance considerations">
  Creating and destroying nodes frequently could impact performance in extreme cases. For games with hundreds of simultaneous sounds, consider pooling AudioStreamPlayer instances instead.
</Accordion>
