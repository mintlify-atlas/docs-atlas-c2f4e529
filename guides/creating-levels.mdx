This guide walks you through creating a new level scene from scratch in Playback, including all required nodes and configuration.

## Level structure overview

Playback levels follow a consistent node hierarchy that includes visual layers, physics, player controls, and recording mechanics. Understanding this structure ensures your levels work seamlessly with the time-rewind system.

<Steps>
  <Step title="Create the base scene">
    Create a new scene with a `Node2D` as the root node. Name it following the pattern `level_X.tscn` where X is your level number.

    ```gdscript
    [node name="Level_1" type="Node2D"]
    ```

    This root node serves as the container for all level elements.
  </Step>

  <Step title="Add visual layers">
    Add parallax backgrounds for visual depth:

    - **ParallaxForground**: For foreground decorative elements
      - Contains `ForgroundParallax` (ParallaxLayer)
        - Contains `ForgroundLayer` (TileMapLayer)
    - **ParallaxBackground**: For background elements
      - Contains `Skybox` (ParallaxLayer) with `motion_scale = Vector2(0.85, 1)`
      - Contains `BackgroundParallax` (ParallaxLayer)
      - Contains `AmbientParallax` (ParallaxLayer)

    Each parallax layer should contain a `TileMapLayer` with appropriate tile sets for visual design.
  </Step>

  <Step title="Add collision layer">
    Create a `TileMapLayer` node named `FloorLayer` as a direct child of the root node:

    ```gdscript
    [node name="FloorLayer" type="TileMapLayer" parent="."]
    tile_set = SubResource("TileSet_8i23i")
    ```

    This layer handles all solid geometry and collision. Make sure your tileset has physics layers configured with collision polygons.
  </Step>

  <Step title="Add UI and player">
    Instance the required scenes:

    ```gdscript
    # UI Canvas
    [node name="CanvasLayer" type="CanvasLayer" parent="."]
    [node name="GameUI" parent="CanvasLayer" instance=ExtResource("game_ui.tscn")]
    player = NodePath("../../Player")
    recording_manager = NodePath("../../RecordingManager")

    # Player
    [node name="Player" parent="." instance=ExtResource("player.tscn")]
    position = Vector2(0, 61)
    ```

    The player should have a `Camera2D` child node for following the player:

    ```gdscript
    [node name="Camera2D" type="Camera2D" parent="Player"]
    offset = Vector2(0, -50)
    zoom = Vector2(5, 5)
    drag_horizontal_enabled = true
    drag_vertical_enabled = true
    ```
  </Step>

  <Step title="Add level completion door">
    Instance the level finish door scene:

    ```gdscript
    [node name="LevelFinishDoor" parent="." instance=ExtResource("level_finish_door.tscn")]
    position = Vector2(450, 64)
    player = NodePath("../Player")
    nextLevel = "res://Scenes/level_2.tscn"
    ```

    Set the `nextLevel` property to point to the next level scene path. Players interact with this door to progress when all objectives are complete.
  </Step>

  <Step title="Add level manager">
    Instance the level manager to coordinate objectives:

    ```gdscript
    [node name="LevelManager" parent="." instance=ExtResource("level_manager.tscn")]
    door = NodePath("../LevelFinishDoor")
    buttons = [NodePath("../Button")]  # For button-based levels
    # OR
    timedObjectives = [NodePath("../TimedObjectiveManager")]  # For timed levels
    ```

    The manager automatically opens the door when all objectives are met.
  </Step>

  <Step title="Add recording system">
    Instance the recording manager and add recording triggers:

    ```gdscript
    [node name="RecordingManager" parent="." instance=ExtResource("recording_manager.tscn")]
    recording_time = 4.0  # Duration in seconds
    ghosts_node = NodePath("../Ghosts")
    player = NodePath("../Player")

    [node name="RecordingTrigger" parent="RecordingManager" instance=ExtResource("recording_trigger.tscn")]
    position = Vector2(224, 72)
    recording_id = 0

    [node name="Ghosts" type="Node" parent="."]
    ```

    The `recording_time` determines how long each recording lasts before auto-rewinding. Create a `Ghosts` node to hold spawned ghost instances during playback.
  </Step>

  <Step title="Add interactable objects">
    Add buttons, crates, or other rewindable objects:

    ```gdscript
    # Buttons for activating doors
    [node name="Button" parent="." instance=ExtResource("button.tscn")]
    position = Vector2(372, 72)

    # Crates (automatically in "rewindables" group)
    [node name="Crate" parent="." instance=ExtResource("crate.tscn")]
    position = Vector2(200, 50)
    ```

    Ensure interactive objects are properly positioned and configured for your puzzle design.
  </Step>
</Steps>

## Recording trigger placement

Recording triggers are placed throughout your level at strategic points. Each trigger:

- Must be a child of the `RecordingManager` node
- Contains a button that players step on to start recording
- Has a unique `recording_id` property
- Position should be accessible but strategic for puzzle design

<Accordion title="Example from level_1.tscn">
  ```gdscript
  [node name="RecordingTrigger" parent="RecordingManager" instance=ExtResource("8_wrinw")]
  position = Vector2(224, 72)
  ```

  This trigger is placed at coordinates (224, 72) in level 1, positioned before the main puzzle area.
</Accordion>

<Note>
  The `recording_id` starts at 0 and should increment for each additional trigger in your level. This ID tracks different recordings separately.
</Note>

## Level completion mechanics

The `LevelFinishDoor` controls progression to the next level:

- Initially locked (closed sprite frame)
- Opened by the `LevelManager` when objectives complete
- Player presses interact key when overlapping the door to advance
- Plays door knock sound if interaction attempted while locked

<CodeGroup>
  ```gdscript level_1 example
  [node name="LevelFinishDoor" parent="." instance=ExtResource("4_7p6j4")]
  position = Vector2(450, 64)
  player = NodePath("../Player")
  nextLevel = "res://Scenes/level_2.tscn"
  ```

  ```gdscript level_2 example
  [node name="LevelFinishDoor" parent="." instance=ExtResource("4_vcilj")]
  position = Vector2(207, 64)
  player = NodePath("../Player")
  nextLevel = "res://Scenes/level_3.tscn"
  ```
</CodeGroup>

<Tip>
  Place the door at the end of your level's physical layout, ensuring players must complete all puzzles to reach it.
</Tip>

## Testing your level

After setting up your level:

1. Verify all node paths are correctly linked (Player, RecordingManager, LevelManager)
2. Test that recording triggers activate when the player steps on them
3. Confirm rewindable objects return to starting positions after rewinding
4. Ensure the completion door opens when objectives are met
5. Check that the next level loads correctly

<Warning>
  Always reference node paths using `NodePath()` relative to the current node. Incorrect paths will cause runtime errors.
</Warning>
